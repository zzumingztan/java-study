# Java Swing 复选框（JCheckBox）实验报告

## 一、实验目的

1. 掌握 Java Swing 中 `JCheckBox` 复选框组件的使用方法。
2. 掌握 `ActionListener` 事件监听器的使用，实现对复选框选择状态变化的响应。
3. 理解 Swing 布局管理器（`BorderLayout`、`FlowLayout`）的基本用法。
4. 掌握 `JTextField` 文本框用于显示动态结果的技术。

## 二、实验环境

- **操作系统**：Windows 11
- **JDK 版本**：Java 8 及以上
- **开发工具**：VS Code / IntelliJ IDEA
- **GUI 框架**：Java Swing（javax.swing）

## 三、实验内容与要求

设计并实现一个 Swing 图形用户界面程序，具体要求如下：

1. 创建一个窗口，标题为"关于复选框"。
2. 窗口中包含三个复选框：**音乐**、**运动**、**上网**。
3. 当用户点击复选框时，在下方文本框中实时显示当前选中的爱好选项。
4. 窗口大小设置为 400×200 像素，启动时居中显示。

## 四、程序设计与实现

### 4.1 整体架构

程序包含一个类 `CheckBoxDemo`，继承自 `JFrame`，并实现 `ActionListener` 接口。类结构如下：

| 成员 | 类型 | 说明 |
|------|------|------|
| `cbMusic` | JCheckBox | "音乐"复选框 |
| `cbSport` | JCheckBox | "运动"复选框 |
| `cbInternet` | JCheckBox | "上网"复选框 |
| `tfResult` | JTextField | 显示选中结果的只读文本框 |
| `label` | JLabel | "爱好选择："标签 |

### 4.2 界面布局

采用嵌套面板的布局方式：

```
┌─────────────────────────────────────────┐
│  JFrame (窗口)                           │
│  ┌─────────────────────────────────────┐│
│  │  topPanel (NORTH) - FlowLayout       ││
│  │  [爱好选择：] [☑音乐] [☑运动] [☑上网]││
│  └─────────────────────────────────────┘│
│  ┌─────────────────────────────────────┐│
│  │  bottomPanel (CENTER) - FlowLayout   ││
│  │  [您选择了：] [  音乐 运动 上网      ]││
│  └─────────────────────────────────────┘│
└─────────────────────────────────────────┘
```

- **外层面板 (`panel`)**：使用 `BorderLayout`，将窗口分为 NORTH 和 CENTER 两个区域。
- **顶部面板 (`topPanel`)**：使用 `FlowLayout`（左对齐），放置标签和三个复选框。
- **底部面板 (`bottomPanel`)**：使用 `FlowLayout`（左对齐），放置标签和结果显示文本框。

### 4.3 核心代码分析

#### （1）窗口初始化设置

```java
setTitle("关于复选框");                              // 设置窗口标题
setSize(400, 200);                                   // 设置窗口尺寸
setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);      // 点击关闭按钮时退出程序
setLocationRelativeTo(null);                         // 窗口在屏幕居中显示
```

#### （2）组件创建与属性设置

```java
cbMusic = new JCheckBox("音乐");
cbSport = new JCheckBox("运动");
cbInternet = new JCheckBox("上网");
tfResult = new JTextField(20);
tfResult.setEditable(false);   // 设置文本框为只读，防止用户手动输入
```

#### （3）事件监听注册

```java
cbMusic.addActionListener(this);
cbSport.addActionListener(this);
cbInternet.addActionListener(this);
```

三个复选框都注册了同一个 `ActionListener`（即当前 `CheckBoxDemo` 对象），实现对点击事件的统一处理。

#### （4）事件处理逻辑

```java
@Override
public void actionPerformed(ActionEvent e) {
    StringBuilder sb = new StringBuilder();

    if (cbMusic.isSelected()) {
        sb.append("音乐 ");
    }
    if (cbSport.isSelected()) {
        sb.append("运动 ");
    }
    if (cbInternet.isSelected()) {
        sb.append("上网 ");
    }

    // 移除末尾多余空格，更新文本框内容
    tfResult.setText(sb.toString().trim());
}
```

**处理流程**：
1. 创建 `StringBuilder` 用于拼接字符串。
2. 依次检查每个复选框的选中状态（`isSelected()` 方法）。
3. 如果选中，则将对应文本追加到 `StringBuilder`。
4. 使用 `trim()` 方法移除末尾空格后，更新文本框显示内容。

#### （5）程序启动

```java
public static void main(String[] args) {
    SwingUtilities.invokeLater(() -> {
        CheckBoxDemo demo = new CheckBoxDemo();
        demo.setVisible(true);
    });
}
```

使用 `SwingUtilities.invokeLater()` 确保 GUI 的创建和显示在**事件调度线程（Event Dispatch Thread）**中执行，保证线程安全。

## 五、技术要点

### 5.1 Swing 线程安全

所有 Swing GUI 操作必须在事件调度线程（EDT）中执行。使用 `SwingUtilities.invokeLater()` 将 GUI 创建代码放入 EDT 执行队列，避免线程安全问题。

### 5.2 布局管理器组合

本程序采用了 `BorderLayout` + `FlowLayout` 的组合布局方式，实现了清晰的面板区域划分。这是 Swing 中常见的布局嵌套模式。

### 5.3 StringBuilder 优化字符串拼接

在事件处理方法中使用 `StringBuilder` 而非直接用 `+` 拼接字符串，提高了字符串操作的效率（虽然在当前场景下性能差异微乎其微，但属良好编程习惯）。

## 七、实验总结

1. **JCheckBox 复选框**是 Swing 中用于多项选择的标准组件，通过 `isSelected()` 方法可以获取当前选中状态。
2. 复选框触发 `ActionEvent` 事件时，可通过实现 `ActionListener` 接口来处理选择状态变化。
3. 合理使用**布局管理器组合**（BorderLayout + FlowLayout）可以快速构建整洁的界面布局。
4. `SwingUtilities.invokeLater()` 是确保 Swing 程序线程安全的标准方式。
5. 实验中涉及的知识点包括：事件监听模型、布局管理、组件属性设置、字符串处理等，是 Java GUI 编程的重要基础。

通过本次实验，我掌握了 Swing 复选框组件的使用方法以及事件处理机制，为后续开发更复杂的 Java GUI 应用程序打下了基础。
